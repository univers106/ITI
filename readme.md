# Contribute

Проект разделён на две части: [frontend](frontend/readme.md) и [backend](backend/readme.md).

Фронтенд компилируется в статические файлы и раздаётся бэкендом.

Структура маршрутов:
- `/api/*` — API бэкенда  
- `/static/*` — все статические файлы, кроме `index.html`  
- `/*` — `index.html` (основной файл фронтенда)

В результате получается веб-приложение, которое в идеале должно кэшироваться в браузере пользователя и выполнять API-запросы для получения данных.

<details>
  <summary>Ветки в репозитории</summary>
  Разработка ведётся в ветке `main`. От неё создаются рабочие ветки, в которых выполняются изменения, после чего они сливаются обратно через Pull Request (все коммиты при этом сквошатся в один).

  
  * `main` - главная ветка (обязательные ПР с ревью)
  * `users/<user_name>/<feature>` - ветки для фичей одного человека (проверяет флоу `branches.yaml`)
  * `features/<feature>` - ветки для совместных фичей (проверяет флоу `branches.yaml`)
</details>
